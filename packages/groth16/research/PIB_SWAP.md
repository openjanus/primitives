# The pi_b Fp2 Swap — A Real Pitfall

This note documents a concrete bug I hit when first trying to verify a Groth16
proof on-chain. The fix is trivial once you know about it, but finding it cost
a few hours of debugging.

## Symptom

Proof generated by snarkJS verifies locally (`snarkjs groth16 verify ...` returns true)
but the on-chain `verifyProof()` call returns `false`. Everything looks correct —
the public signals match, the proof is fresh, the verifier contract is the right one.

## Root cause

snarkJS and EIP-197 use opposite orderings for Fp2 element components.

An Fp2 element `a + b*i` (where `i` is a specific root in the extension field) is:
- snarkJS: encoded as `[a, b]` (real part first)
- EIP-197: encoded as `[b, a]` (imaginary part first)

So `pi_b` from snarkJS:
```json
{
  "pi_b": [
    ["real_0", "imag_0"],
    ["real_1", "imag_1"]
  ]
}
```

Must be passed to the EVM verifier as:
```javascript
pB = [
  [proof.pi_b[0][1], proof.pi_b[0][0]],  // [imag_0, real_0]
  [proof.pi_b[1][1], proof.pi_b[1][0]],  // [imag_1, real_1]
]
```

## Why only pi_b?

`pi_a` and `pi_c` are G1 points (over Fp, not Fp2), so no swap is needed.
Only `pi_b` is a G2 point (over Fp2) and therefore affected.

The snarkJS-generated Solidity verifier expects the swap to be applied
by the caller before passing `_pB` to `verifyProof()`.

## Verification

In the `circuit_test.mjs` reference implementation:
```javascript
// snarkjs proof.pi_b = [[re0, im0], [re1, im1]]
// EVM expects:          [[im0, re0], [im1, re1]]
const pB = [
  [proof.pi_b[0][1], proof.pi_b[0][0]],
  [proof.pi_b[1][1], proof.pi_b[1][0]],
];
const onChainValid = await verifier.verifyProof(pA, pB, pC, pub6);
// Returns true ✓
```

Without the swap, `verifyProof` returns false — no revert, no error message.

## SDK fix

The `@openjanus/groth16` SDK's `proofToEVMCalldata()` function applies the swap
automatically. If you use the SDK, you don't need to think about this.

```typescript
import { proofToEVMCalldata } from "@openjanus/groth16";

const { pA, pB, pC } = proofToEVMCalldata(snarkjsProof);
// pB already has the swap applied — safe to pass to verifyProof()
```
